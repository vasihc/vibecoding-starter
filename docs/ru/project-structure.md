# Структура проекта

Структура папок проекта организована следующим образом:

## Основные директории

- `pages`: Содержит страницы Next.js для серверного рендеринга.
- `components`: Содержит переиспользуемые React-компоненты.
- `lib`: Включает утилитарные функции и модули.
- `public`: Хранит статические ресурсы, такие как изображения, шрифты и таблицы стилей.
- `styles`: Содержит глобальные стили и конфигурацию Tailwind CSS.
- `firebase`: Содержит конфигурацию Firebase и связанные функции.

## Пример расширенной структуры

Вы можете изменять или расширять структуру папок в соответствии с потребностями вашего проекта.
Например, дополнительные подпапки могут быть организованы так:

```
components/
├── UI/
├── custom/
├── partials/
└── ...

public/
├── assets/
│   ├── images/
│   ├── fonts/
│   └── gifs/
└── ...
```

## Структура App Router (Next.js 13+)

- **`src/app/`** - Страницы и макеты App Router
  - `layout.tsx` - Корневой макет, оборачивающий все страницы в AuthContextProvider
  - `page.tsx` - Главная страница
  - `signin/`, `signup/` - Страницы аутентификации
  - `admin/` - Защищенный маршрут администратора

## Интеграция Firebase

- **`src/firebase/`** - Слой интеграции Firebase
  - `config.ts` - Инициализация Firebase с использованием переменных окружения
  - `auth/` - Функции аутентификации (signIn.ts, signup.ts)
  - `firestore/` - Операции с базой данных (addData.ts, getData.js)

## Управление контекстом

- **`src/context/`** - React контексты
  - `AuthContext.tsx` - Глобальное управление состоянием аутентификации с использованием Firebase Auth

## Библиотека компонентов

- **`src/components/`** - Переиспользуемые компоненты
  - `ui/` - Компоненты shadcn/ui (Button, Card, Dialog и др.)

## Утилиты

- **`src/lib/`** - Утилитарные функции
  - `utils.ts` - Помощник cn() для объединения className
