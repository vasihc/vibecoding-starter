# AI Dev Tasks

Структурированный рабочий процесс для разработки с помощью AI, используя markdown-промпты. Разбивайте сложные функции на управляемые пошаговые задачи с встроенными контрольными точками проверки.

## Основной рабочий процесс

### 1. Создание PRD
Используйте `@plan/create-prd.md` для генерации документа требований к продукту:
```
Use @plan/create-prd.md
Вот функция, которую я хочу создать: [Опишите вашу функцию]
```

### 2. Генерация задач
Используйте `@plan/generate-tasks.md` для разбивки PRD на выполнимые задачи:
```
Теперь возьми @MyFeature-PRD.md и создай задачи используя @plan/generate-tasks.md
```

### 3. Обработка задач
Используйте `@plan/process-task-list.md` для последовательной работы над задачами:
```
Пожалуйста, начни с задачи 1.1 и используй @plan/process-task-list.md
```

### 4. Проверка и одобрение
- Проверяйте каждую выполненную задачу
- Отвечайте "yes" для одобрения и перехода к следующей задаче
- Предоставляйте обратную связь, если нужны изменения

## Файлы

- **`plan/create-prd.md`** - Генерация документов требований к продукту
- **`plan/generate-tasks.md`** - Разбивка PRD на выполнимые списки задач
- **`plan/process-task-list.md`** - Пошаговая обработка задач с контрольными точками одобрения

## Преимущества

- **Структурированная разработка** - Четкий процесс от идеи до кода
- **Пошаговая проверка** - Проверка и одобрение каждого изменения
- **Управление сложностью** - Разбивка больших функций на управляемые задачи
- **Повышенная надежность** - Более надежно, чем единый большой промпт
- **Отслеживание прогресса** - Визуальное представление выполненной работы

## Советы по использованию

- Будьте конкретны в описании функций
- Используйте мощные AI-модели для лучших результатов
- Разбивайте задачи дальше, если AI испытывает трудности
- Направляйте и итерируйте по мере необходимости
